# Cultural Discrimination in Large Language Models

Code and data for the paper:

> **Cultural Discrimination in Large Language Models: Measuring and Mitigating Bias Using the World Values Survey**
> Antoni Czolgowski, University of Colorado Boulder
> OSSConf 2026

## Summary

We measure cultural bias in three open-weight LLMs by comparing their simulated survey responses to real human data from the World Values Survey (Wave 7, Q164: "How important is God in your life?"). Each model is prompted with 63 demographic personas (3 countries x 2 sexes x 4 age groups x 3 education levels) and queried 100 times per persona. Bias is quantified using the normalized Wasserstein-1 (W1) distance between model-generated and human response distributions.

**Key findings:**
- All three models show significant cultural bias (ANOVA F = 42.8, p < 0.001)
- Models are most accurate for their culturally aligned country
- LoRA fine-tuning on the 5 worst-case personas reduces bias by 16.8% for Bielik but increases it by 13.9% for Qwen
- Fine-tuning side effects are model-dependent: Bielik improves broadly, Qwen worsens on non-target personas

| Model | Origin | CHN W1 | SVK W1 | USA W1 |
|-------|--------|--------|--------|--------|
| Gemma 3 12B | Google (USA) | 0.251 | 0.137 | **0.067** |
| Bielik 11B v3 | SpeakLeash (Poland) | 0.341 | **0.080** | 0.192 |
| Qwen 3 4B | Alibaba (China) | **0.081** | 0.196 | 0.252 |

*Bold = lowest W1 (best accuracy) per model. Lower is better.*

## Pipeline

| Step | Script | Language | Description |
|------|--------|----------|-------------|
| 1 | `01_wvs_data_prep.R` | R | Extract and validate Q164 data for CHN, SVK, USA |
| 2 | `02_human_distributions.R` | R | Compute per-persona PMFs (63 personas) |
| 3 | `03_run_baseline.py` | Python | Query 3 LLMs x 63 personas x 100 repetitions |
| 4 | `04_baseline_analysis.R` | R | W1 distances, bootstrap CIs, ANOVA (RQ1) |
| 4b | `04b_training_data.R` | R | Generate LoRA training data from full WVS |
| 4c | `04c_question_mapping.R` | R | Join question text and scales to training data |
| 5 | `05_lora_finetune.py` | Python | LoRA fine-tuning on worst-case personas |
| 6 | `06_run_posteval.py` | Python | Re-query fine-tuned models on all 63 personas |
| 7 | `07_posteval_analysis.R` | R | RQ2 + RQ3 paired t-tests, Bonferroni correction |
| 8 | `08_figures.ipynb` | Python | Generate all publication figures |

Supporting modules (imported by pipeline scripts):
- `config.py` — shared configuration and paths
- `parser.py` — response parser (1-10 extraction)
- `models.py` — Ollama and Transformers model wrappers
- `experiment.py` — experiment class with checkpointing

## Data

Steps 1, 4, 4b, and 4c require the full WVS Wave 7 microdata, available at [worldvaluessurvey.org](https://www.worldvaluessurvey.org/WVSDocumentationWV7.jsp) (free registration). Set `export WVS_CSV_PATH="/path/to/WVS_Cross-National_Wave_7_csv_v6_0.csv"` before running those steps. All other data is included in the repository.

## Environment Setup

### Python (3.10+)

```bash
pip install -r requirements.txt
```

### R (4.3+)

```r
install.packages("tidyverse")
```

### Models

- **Gemma 3 12B**: served via [Ollama](https://ollama.com) (`ollama pull gemma3:12b`)
- **Bielik 11B v3** and **Qwen 3 4B**: downloaded from HuggingFace, loaded with 4-bit BitsAndBytes quantization

## HPC Setup

For running on a SLURM cluster (tested on CU Boulder Alpine with A100 80GB GPUs):

```bash
export LLM_BIAS_DIR="/path/to/llm-cultural-bias"
export LLM_BIAS_MODELS_DIR="/path/to/models"
export SLURM_ACCOUNT="your_account"

# Test run (~1 hour)
sbatch slurm/lora_test.sh

# Full training (~4 hours)
sbatch slurm/lora_full.sh
```

Hardware requirements: NVIDIA GPU with at least 24 GB VRAM (A100 recommended). Each model uses approximately 6-8 GB with 4-bit quantization.
